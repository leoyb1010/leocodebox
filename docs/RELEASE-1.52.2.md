# leocodebox 1.52.2 — 驾驶舱首屏加载提速

1.52.1 修好了驾驶舱的数据接线,但首屏仍要等好几秒才填上——瓶颈定位为「Claude 窗口消耗」卡:它每次都全量扫描本地会话日志。本版把该扫描从 ~9 秒降到 ~1.4 秒(冷)/ 0 毫秒(缓存命中)。纯性能修复。

## 修了什么

**Claude 窗口消耗扫描过慢**——`estimateClaudeQuota` 之前每次调用都遍历 `~/.claude/projects` 下**全部** jsonl(本机 20,140 个文件 / 2.1GB),且无缓存,导致该卡片阻塞 ~9 秒,还每 30 秒轮询重扫一次。改为:
- **按 mtime 剪枝**:最后修改时间早于 7 天窗口的文件其内容必然全部过期,直接跳过(本机由 20,140 个降到 4,111 个)
- **并发读取**(并发 16)剩余文件
- **60 秒结果缓存**(长于前端 30 秒轮询),轮询不再重扫

实测:冷扫描 ~1.4s、缓存命中 0ms、统计数值不变(5h/7d IO tokens 与旧版一致)。

## 验证
- typecheck 0、lint 0、client 71/71、server 299/299
- 直接调用 `estimateClaudeQuota()` 实测冷 1417ms / 热 0ms;安装后首屏各卡片秒级填充
