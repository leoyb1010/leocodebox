# leocodebox 1.55.0 — 网关请求级路由 + mid-session 故障转移(融合第二步)

在 1.54 的「本机计量网关」地基上,把网关从「会话级转发」升级成**请求级路由 + 自动故障转移**:每个请求在网关侧按当前 active 节点/槽位**实时解析**,不再钉死在会话启动那一刻选中的节点。切换节点、改槽位绑定,**下一个请求就生效(mid-session)**;某个节点 429/5xx/网络故障时,网关在**吐出任何字节之前**自动切到同 target 的兄弟节点,客户端永远看不到半截响应。

## 多了什么
- **请求级路由(mid-session)**:CLI env 注入的令牌从 `lgw:<节点id>` 改为 `lgw:<target>[:<槽位>]`。网关每次请求现查 active 绑定 → 会话进行中换节点/换槽,**无需重启会话**即刻跟随。
- **自动故障转移(容错)**:主节点返回可重试错误(429 / ≥500 / 连接失败)且**尚未开始流式输出**时,按 [主节点 → 同 target 兄弟节点] 顺序依次重试;每次尝试独立计量;全部失败才 fail-closed 返回真实错误。**绝不**把两个节点的输出拼接给客户端。
- **化繁为简 · 驾驶舱可见**:网关卡在「实时计量中」那行右侧,新增两枚极简徽标——`N 节点分流`(本窗口内实际分流到的节点数)与 `M 次容错`(触发的故障转移次数)。**只在真的发生分流/容错时才出现**,单节点无噪声时保持安静。派生自既有计量,无新增状态。

## 安全与边界(不变)
- **默认关**;不开启行为与 1.53 完全一致。
- **忠实透传**:响应逐字节 pipe,计量只读 tee 副本;故障转移只发生在**首字节之前**。
- **loopback-only**;`lgw:` 令牌鉴权;真实 key 不进 CLI env。
- **fail closed**:整条链失败即返回最后一个真实错误,绝不静默改写。

## 验证
- 门禁全绿:typecheck 0、ESLint 0、client 71/71、server **310/310**、生产构建通过。
- 新增单测:令牌解析(x-api-key / Bearer / 拒非网关令牌)、target 令牌解析出「active 优先 + 兄弟兜底」有序链、槽位令牌解析出「绑定节点优先」、钉死 providerId 单节点不故障转移、未知令牌空链;计量路由信号(节点数 / 容错次数 / 窗口)。
- 纯函数 `selectUpstreamChain` 覆盖路由+故障转移排序,不依赖网络。
