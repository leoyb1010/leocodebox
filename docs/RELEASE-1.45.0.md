# leocodebox 1.45.0

「资产可操作」——把 1.44 刻意留白的 MCP / Skill 管理交互补齐:全景板上直接点 CLI 标签装/卸,技能一键软删除并可在回收站找回。全程门禁全绿 + 对抗式审查(修 2 个确认缺陷)。

## 多了什么

**MCP 全景板:每 CLI 一键装/卸**
- 「设置 → MCP」的跨 CLI 全景板从只读徽标升级为可点 chip:某服务器在哪个 CLI 已装就显示实心(点一下从该 CLI 移除,hover 变红提示);没装的显示虚线(点一下把配置复制装进去)。一处纵览、一处操作,不用再逐个 CLI 来回切。
- 目标 CLI 不支持该传输(如 SSE 只有 Claude 支持,Cursor/Codex/OpenCode 不支持)的 chip 自动置灰并解释原因——不给注定失败的死路按钮。
- 托管的 `cloudcli-` 服务器保持只读(leocodebox 自己写的,不该手删)。
- 写入走服务端事务化备份,坏写不会污染配置。

**Skill 一键删除 + 回收站找回**
- 用户级技能卡右上角新增删除按钮(仅对用户自己的、非插件技能开放)。删除是软删除——移进 `~/.leocodebox/trash`,不是 `rm -rf`。
- 「设置 → 关于」新增回收站板块:列出已删技能可一键还原,并只读展示最近的配置备份(名称/大小/时间,不泄露绝对路径)。

## 更稳的

- 对抗式审查修掉 2 个确认缺陷:
  - **Skill 删除空操作假成功**:含空格/点/双横杠的真实文件夹名被服务端二次归一化后解析到不存在路径,`removed:false` 却回 `success:true`——用户看到「已移入回收站」但技能纹丝不动。改为按精确名查找(路径越界守卫仍在),客户端补防御:空操作视为失败,绝不谎报回收站条目。
  - **MCP 死路 chip**:不支持该传输的 CLI 仍渲染可点「+安装」,点了必被后端 400 拒绝。改为传输不匹配即置灰禁用。
