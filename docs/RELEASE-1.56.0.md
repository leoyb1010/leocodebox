# leocodebox 1.56.0 — 网关上下文瘦身 · 降本(融合第三步)

融合 pi 的 compaction 思路,落在我们唯一能看到每个请求的地方——本机网关。开启后,当一次 Claude `/v1/messages` 请求带着很长的历史时,网关在**转发前**把**历史里超大的 `tool_result`(工具输出)**就地裁剪,直接减少输入 token(降本)。**确定性裁剪,不做模型调用,零额外延迟/成本**;只减 token,不改语义结构。

## 多了什么
- **上下文瘦身(网关子开关,beta,默认关)**:网关卡片新增一枚小开关。开启后仅在 `/v1/messages` 且历史足够长时触发。
- **正确性优先的裁剪不变量**(所以敢作为 beta 暴露):
  - **系统提示永不动**;
  - **首轮(任务设定)原样保留**;
  - **最近 20 轮原样保留**;
  - 只裁 **历史中间段的 `tool_result`**(工具输出),**text / tool_use 一律不动**;
  - 只有历史明显很长(> 25 轮)才动手,否则完全 no-op;
  - 任何解析失败 / 非预期结构 → **原样放行**(fail-open,绝不改坏一个读不懂的请求)。
- **化繁为简 · 可见**:卡片显示「已省 ~N 字符」实时累计;一个小开关即开即关,不新增页面、不加复杂度。

## 安全与边界
- **默认关**;不开启行为与 1.55 完全一致。裁剪有损(按设计),因此 opt-in。
- **只减不增**:被裁的旧工具输出替换为一行中文标记,近 20 轮与首轮 + 系统提示完整,当前任务上下文不受影响。
- 与忠实透传/请求级路由/故障转移正交:瘦身只发生在**转发前**、仅对 Messages 写入;其余请求原样。

## 验证
- 门禁全绿:typecheck 0、ESLint 0、client 71/71、server **314/314**、生产构建通过。
- 新增单测(纯函数,不依赖网络):只裁旧的超大 tool_result 且系统/首轮/近段/文本块均保留、savedChars 精确、输入对象不被就地改写;低于阈值 no-op(同引用);`compactRequestBody` 长体裁剪 + 坏 JSON/无可裁 → fail-open 返回 null;累计统计正确。
