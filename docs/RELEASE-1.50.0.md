# leocodebox 1.50.0 —「指挥部」2.0(1–5 级)

把工作台升级为「并行调度智能体舰队的本地指挥部」。本版落地 2.0 计划 1–5 级的**后端能力 + REST API + 测试**,并交付 **L4 任务看板** 前端旗舰界面。全程本地优先、无新增网络面、向后兼容。

## 新增能力

**L1 · 模型弹药库**——40+ 模型的 context/输出上限/输入输出单价/视觉·推理·工具能力/wire 协议(离线嵌入)。用量成本与上下文窗口改由弹药库精确驱动(用户价目覆盖 > 弹药库 > 旧表 > 0,未知不编造)。

**L2 · 路由控制面(Leoapi 2.0)**——场景槽位(default/background/longContext/think)把任务路由到不同 Leoapi 节点,**零常驻代理**(复用 per-session env 接管)。未绑定槽 = 旧单活跃行为。

**L3 · 舰队(worktree 并行)**——每个任务独立 git worktree(`lcb/<slug>`),会话绑定后以 worktree 目录为 cwd 并行互不踩脚;`git merge-tree` 无副作用冲突预览、脏树拒删。

**L4 · 任务看板(含界面)**——左侧栏新增「任务」视图:四列看板 + 建卡 + 开工(自动起 worktree、按档案定 provider、自动选路由槽)+ 送审/完成(冻结成本)/重试/丢弃。状态机驱动舰队。

**L5 · 登录态快照**——claude/codex 官方账号登录态命名快照与一键切换(原子写、先备份后覆盖、0600、防路径穿越),与 Leoapi 互补。

## 验证

- 门禁全绿:客户端/服务端 typecheck、ESLint 0 警告、生产构建;测试 **393**(desktop 27 + client 71 + server 295,本轮新增 25 条覆盖 L1–L5)。L3/L4 用真实临时 git 仓库端到端验证。
- **对抗式审查**(10 agent 并行 + 逐条对抗验证):0 blocker;5 处 major 全部修复(含用量覆盖遮蔽、悬挂路由槽掉本机环境、合并预览误判、看板通用转移绕过副作用),每处附回归测试。
- 去死码/化繁为简:路由启发式接入起卡路径、删无人消费的重导出与死类型、合并重复配置读取、原子凭据写入。
- 真机(已签名正式包)核对任务看板界面与建卡/开工流程。
