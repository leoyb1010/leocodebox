# leocodebox 1.49.6

热修复:1.49.5 的 spawn EBADF 修复不彻底——**应用刚启动的头几秒**探针仍会瞬态失败,导致「本机智能体」在启动后第一次加载时把已装的 CLI 全部显示为「未安装/未找到本机命令」。

## 修了什么

**[严重] 启动初期本机智能体全部显示「未安装」**
- 现象:早晨/重启后打开 App,设置里 7 个 CLI 全部提示「未找到本机命令」,但机器上明明都装了。
- 根因:`spawn EBADF` 是打包 GUI 应用里 libuv 的**瞬态 fd 竞态**,在 Electron 启动后的头几秒最严重——此时**连完全串行的 spawn 也会随机失败**(1.49.5 的串行门只解决了稳态并发,压不住启动窗口期)。启动后第一拉状态时 `which` 本身 EBADF → 全部判成"未找到";10 秒后同样的探针全部正常。已在真机复现确认。
- 修复:把 EBADF 视为「稍后重试」而不是「未安装」——探针失败且错误是 EBADF 时自动重试(最多 5 次、300ms 起步递增退避),叠加 1.49.5 的 stdin ignore + 串行门。
- 真机验证:**连续 3 轮冷启动、服务起来的第 0 秒立即拉取状态,7 个 CLI 全部识别、0 错误**(修复前同一时点会大面积 EBADF)。

## 验证

- 门禁全绿:typecheck、ESLint 0 警告、生产构建;测试 366(desktop 27 + client 70 + server 269)。
- 真机(已签名正式包、访达双击启动):3× 冷启动 t+0 立即拉取,7/7 识别、无 EBADF。
