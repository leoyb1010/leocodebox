# leocodebox 1.51.0 —「接线」

1.50 把「指挥部」1–5 级的后端能力全部建好,但用户可感知的端到端闭环断在最后一公里。1.51 不加新功能,只把 1.50 已承诺的卖点接通,并顺手加固凭证落盘安全。全程本地优先、无新增网络面、向后兼容。

## 多了什么(接线)

**看板「开工」真的会开工**——起卡后自动跳进该任务的会话,并把卡片目标(goal)作为首条 prompt 经 WebSocket 发出。此前起卡只建数据库行、目标从不发给智能体,卡片停在原地。

**看板「丢弃」不再必然 404**——前端与后端的 discard 协议对齐(改发 POST),脏 worktree 会先确认再删。

**L4 闭环 · 合并完成**——待审卡片新增「合并完成」:把任务 worktree 合并回基线分支再标记完成。服务端先做无副作用冲突预览、有冲突则拒绝(绝不强合),冲突文件清单直接回显。

**看板 ↔ 聊天双向跳链**——进行中/待审卡片加「打开会话」按钮,一键回到任务正在跑的会话。

**Settings 语音页可达**——修复标签归一化白名单漏配 'voice',此前进语音设置会被打回「智能体」页。

## 更稳的(安全加固)

**外部凭证落盘加密**——GitHub 等 `user_credentials` 从此以 AES-256-GCM 加密存储,启动时一次性迁移历史明文行。数据库被拷走不再直接泄露可用令牌。

**统一密钥盒**——新增 `server/shared/secret-box`,凭证 / JWT 签名密钥 / browser-use MCP token 共用一套加密与密钥体系。

**收紧密钥派生**——server 独立运行(非桌面壳)时,加密密钥改用持久化随机密钥(0600),替代此前 `sha256(username:homedir)` 的同机可推导派生;解密兼容旧密钥并在下次写入时自动升级。桌面壳路径继续走 safeStorage/keychain。

## 刻意没做的(诚实标注)

- **cookie + WS 鉴权改造(#9)**:全权静态 token 裸存 localStorage 的 RCE 链路属实,但修复是同时改 apiClient 与 WS 鉴权的大重构,需独立版本谨慎落地,不塞进本版热更。当前风险以「先有 loopback XSS 前提」为门槛。
- **L1 弹药库上墙 / L2 路由表 UI / L5 登录快照进新建会话**:1.50 已具备 API,界面接线留给 1.52「看见」。
- **task-master 统一、看板拖拽动效、base.css 色值 token 化、i18n 44 key 补齐、桌面三缺(自启/深链/系统代理)、win latest.yml 发布链**:分别属于 1.52/1.53,本版不碰,避免半成品上热更。
