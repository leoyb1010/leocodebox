# leocodebox 1.49.7

热修复:1.49.6 遗留的一串「本机智能体」问题一次收口——读取慢(30-60 秒)、点更新无反应、认证状态检查失败、EBADF 毒化快照把已装 CLI 全部误报成未安装,外加账户页大色块和绿底绿字徽章两处视觉问题。

## 修了什么

**[严重] 本机智能体读取 30-60 秒 + 点更新无反应**
- 1.49.6 的全串行探针门把 ~28 个探针串成一条队(实测 60s+),更新请求也被堵在队里。
- 改为:**磁盘快照秒开**(打开面板 0.03s 显示上次已验证的状态)+ 后台单飞刷新(并发 4,约 6s)+ 前端静默换新。实测更新按钮 5s 完成(opencode 1.18.2→1.18.3 真机验证)。

**[严重] EBADF 毒化:点更新后所有 CLI 变「未安装」**
- `spawn EBADF` 在 npm 安装窗口内可持续很久,超过重试预算;坏结果一度写进快照,把 7 个已装 CLI 全部误报成未安装。
- 修复原则:**EBADF 只能算「未知」,永远不允许把好状态降级**——
  - `which` 级 EBADF 抛 `TransientProbeError`,不解释为「未找到」;
  - 快照写入前逐工具做「最后好状态」合并(EBADF 结果不覆盖已验证条目,附单测);
  - 安装/更新 handler 的前后探测在瞬态失败时回落快照,不再误判「not installed」拒绝更新。

**[严重] 账户页「认证状态检查失败」(claude/cursor/codex 全报)**
- 认证/版本探针(`cli-version.util`)没有 EBADF 重试,启动窗口一撞就失败。补上同款重试(≤5 次退避)。真机验证:三家均显示真实登录态,无 EBADF。

**视觉两处**
- 账户卡大色块(Claude 亮蓝/Cursor 亮紫)全部换成与设置一致的中性卡片(身份用 logo 表达)。
- 全库修复「绿底绿字」徽章(`bg-success text-success` 同色叠同色,14 个文件):改为浅色底 + 主题色字,明暗两版可读。
- Gemini CLI / Hermes Agent 药丸点击增加滚动定位 + 高亮闪烁反馈(它们没有账户页,原来点了没可见反应)。

**代理共存核查(应约)**
- 实证:App 仅监听 127.0.0.1 的 38473 与两个 53xxx 临时端口,未触碰 7890-7897 代理段;App 运行期间经 Clash(7897)的请求 0.24s 正常返回。不存在抢占代理端口。

## 验证

- 门禁全绿:typecheck、ESLint 0 警告、生产构建;测试 367(desktop 27 + client 70 + server 270,含「EBADF 不覆盖好状态」单测)。
- 真机(热替换的正式包、访达启动):冷启动面板 0.03s 秒开、7/7 识别;毒化快照清除后重建正确;更新按钮 5s 完成且快照同步;认证状态三家真实;设置页截图核对配色与药丸。
