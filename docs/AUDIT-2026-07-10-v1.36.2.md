## 已修复的发布阻断问题

1. 桌面进程从 Finder 启动时无法发现用户 CLI：合并登录 Shell、包管理器和常见用户目录，并把检测到的 Claude/Codex 可执行文件传给真实 SDK 运行器。
2. 打包后 `better-sqlite3` ABI 不一致：发布流程统一按 Electron Node ABI 重建和验收。
3. 外置 `CODEX_HOME` 等目录备份恢复到错误位置：备份记录真实目标路径，恢复只允许当前已识别的 Agent 配置路径。
4. Gemini `.env` 写入破坏注释和未知变量：只更新受管键，保留原格式与其他配置。
5. OpenCode 数据库变更只同步一个会话：单次 watcher 事件同步全部活动会话，并把同步错误上抛。
6. 会话创建可绕过项目路径校验：新会话只能绑定已注册且未归档的本地项目。
7. Codex/OpenCode 导入选错活动 Provider：按 `model_provider` 和模型前缀选择真实活动项。
8. 401/403 被显示为绿色成功：凭据拒绝与网络可达分开呈现。
9. Leoapi 窄窗口隐藏设置：390px 宽度仍可访问刷新、设置、导入和恢复。
10. Leoapi 固定深色且无法返回：增加明确返回按钮，与主界面共享“跟随系统 / 浅色 / 深色”。
11. Leoapi 只有单地址和单次连通检测：增加多地址测速、最快地址选择、模型列表读取、真实模型测速和检测参数。
12. Claude 第三方接口缺少模型角色映射：分别写入 Sonnet、Opus、Haiku 和默认模型。
13. 更新凭据向后端子进程泄漏：GitHub 更新 token 只留在 Electron 主进程，子进程显式移除。
14. 本地页面短暂显示登录页：本地 token 准备期间显示加载状态，失败时显示本机服务错误而非账号密码表单。

## 残余风险

1. GitHub 私有仓库的应用内更新首次需要用户配置只读 token；v1.36.1 没有更新器，因此必须手动安装一次 v1.36.2，后续才可热更新。
2. 打包仍携带较完整的 Node 依赖，DMG 体积偏大；这是体积问题，不影响离线启动或 CLI 发现。
3. 可选桌面自动化依赖链存在 7 个 moderate 漏洞且无可用修复；发布包不向公网暴露对应能力。
4. Provider 模型测速会发送最短真实请求，界面已明确提示可能产生极少量计费，并且不会自动在后台执行。
