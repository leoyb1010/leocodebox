# LeoJarvis 能力并入 leocodebox 分析

日期：2026-07-10

## 结论

LeoJarvis 不应整体并入 leocodebox。leocodebox 的产品边界是“本机编码 Agent 工作台”，只吸收能强化项目连续性、会话复用和开发交接的能力；情报、扫描、天气、RSS、个人生活记录、系统守护和个人 OS 首页不进入本产品。

## 现在适合并入

1. **继续工作首页**：把最近项目、最近 Agent 会话、未完成任务和异常运行集中成可直接继续的工作列表。数据必须来自本机项目与会话索引，不生成新的云端状态。
2. **项目笔记**：每个项目提供 `草稿 / 决策 / 交接` 三类本地 Markdown 笔记，并支持从当前会话显式执行“保存为项目笔记”。
3. **会话交接摘要**：从选定会话生成可编辑的交接内容，写入项目目录或 `~/.leocodebox/notes/`；必须由用户确认，不做后台自动记忆。

## 后续评估

1. **项目 Research Sets / Notebook**：仅关联当前项目的本地文件、URL 摘要和已选会话，不建立跨项目知识池。
2. **确认式项目记忆**：Agent 提议保存事实，用户逐条确认后写入项目级文件；默认不自动召回跨项目内容。

## 不应并入

- 情报流、全网扫描、RSS、天气和个人生活日志。
- 系统资源守护、通用个人助手首页和与编码 Agent 无关的快捷入口。
- 自动抓取全部会话并形成不可审计的长期记忆。
- 跨项目自动召回、未经确认的提示词注入和来源不明的附件知识库。

## 必须保留的安全边界

- 项目记忆按真实项目路径隔离，移动或重命名时显式迁移。
- 每条笔记记录来源会话、创建时间和最后编辑时间。
- 外部内容进入 Agent 上下文前标记为不可信数据，不能把网页指令当系统指令。
- 附件限制类型、数量和大小；路径必须限制在当前项目或 leocodebox 上传目录。
- 删除、覆盖、跨项目移动和自动注入上下文都需要明确确认。

## 建议顺序

先实现“继续工作首页 + 项目笔记 + 保存会话为笔记”，验证重度 Agent 用户是否持续使用；再决定是否增加 Research Sets。不要在当前跨设备 CLI、签名公证和热更新稳定性完成前扩展产品边界。
