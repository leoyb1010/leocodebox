# leocodebox 1.58.0 — 自有内核 · 交互式运行卡(pillar 3 · UI 化繁为简)

给 1.57 的内核运行时加上**驾驶舱入口**:一张 cool-simple 的卡,让你真正用上自有内核 —— 填根目录 + 提问 + 运行,内核只读读文件后作答,并展示精简的工具轨迹。复用 1.57 已发布的 `/kernel/run` 端点,不新增后端能力面。

## 多了什么
- **自有内核卡(驾驶舱右栏,beta)**:两个输入(根目录 / 问题)+ 一个运行按钮。`⌘↵` 直接运行。
- **回答 + 精简轨迹**:展示内核的最终回答、用到的工具(read_file / list_dir 小徽标)、步数;触及步数上限会标注「已达上限」。底部标注实际服务节点/模型。
- **化繁为简**:一张卡装下"提问→内核读码→作答"的完整闭环;无多余面板、无复杂配置。沿用统一 motion/radius token 与既有卡片视觉,风格一致。
- **无活跃节点时诚实提示**:没有 active claude 节点时,卡片直接显示后端返回的友好错误(请先激活一个节点),不转圈、不臆造。

## 安全与边界(承 1.57)
- 后端仍是 **只读 + 根目录沙箱 + 步数上限**;卡片只是入口,不放宽任何权限。
- 复用本地 token 鉴权的 `/kernel/run`;真实 key 不进前端。

## 验证
- 门禁全绿:typecheck 0、ESLint 0(含 design-system radius/motion 规则)、client 71/71、server **320/320**、生产构建通过。
- 装机后:驾驶舱可见内核卡;端点校验(空 prompt / 非法根目录 400)、无活跃节点 409 均按预期(实测)。
