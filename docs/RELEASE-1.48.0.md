# leocodebox 1.48.0

「更稳的切换」——Leoapi 接口切换的可靠性升级(cc-switch v3.17 对比后的免代理低垂果实):后台健康轮询、可选自动故障转移、切走前回收手改。不引入任何新常驻进程。全程门禁全绿 + 对抗式审查(18 agent 逐条证伪,修 15 个确认缺陷)。

## 多了什么

**后台健康轮询(设置 → 健康监控)**
- 每 5 分钟(可调 1-60)自动探测各智能体当前启用的节点。探测走**免费的模型列表请求,永不消耗 token**;中转站不实现 /models 也不误判(404 视为存活,只有凭据被拒/5xx/网络错误算失败)。
- 连续 2 次失败 → 节点降级:列表顶部琥珀横幅提醒 + Doctor 健康灯变琥珀(带失败详情),并提供「切换到健康备用」「立即重测」。

**配置级自动故障转移(每智能体可选,默认关)**
- 勾选的智能体在当前节点降级时,自动对备用节点**逐个活体实测**,切换到第一个探测通过的——走与手动切换完全相同的事务化写入(先备份)。
- 切换后蓝色横幅告知「A → B」,**一键撤销**切回原节点;30 分钟自动切换冷却,两个半坏节点不会来回震荡改写你的配置。
- 横幅的手动「切换到健康备用」同样走活体实测(不用可能过期的缓存排名),显式点击即同意、不受冷却限制。

**切走前回收手改**
- 你手动改过 live 配置(如换了 API key、调了模型)再切换节点时,这些手改会先收进当前节点的记录——切回来恢复的是你今天真实在用的配置,不是过期快照。仅当 live 配置仍指向同一目的地才收编,其他工具改写过的配置绝不污染记录。

**shell 覆盖检测扩展**
- claude 补充 AUTH_TOKEN/API_KEY 检测;新增 gemini(终端是 Gemini CLI 唯一会话,shell 变量静默击败 `.env` 在此最致命);同 URL 导出不误报。

## 更稳的(对抗式审查修 15 处,1 高 6 中 8 低,0 误报)

- **[高] 自动切换也回收手改**:否则手改的 key/模型被静默覆盖、撤销恢复出过期配置;且「live 换新 key、记录还是旧 key」会 401→降级→把可用配置盖掉。
- **[中] 候选探测移出全局锁**:原先降级时 8s×N 的串行探测会让所有手动操作排队卡死。
- **[中] 锁内复核设置**:探测进行中关掉自动切换,不再被"迟到的切换"改配置。
- **[中] 30 分钟震荡冷却**(见上)。
- **[中] 坏 baseUrl 不再炸整轮探测**(逐 target 隔离,其余照常)。
- **[中×2] 设置未加载时禁止保存/渲染默认值**——不再把表单默认值写回冲掉真实设置;并发时快照带真实持久化设置。
- **[低×8]**:探测后释放连接(keep-alive 不再慢性泄漏)、codex 收编补 base_url 守卫(key 不会发错主机)、面包屑不被旧副本复活、设置接口错型字段忽略不重置、UI 候选宁缺毋滥、gemini 告警同 URL 去重等。
