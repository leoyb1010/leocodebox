## 多角色对抗

### 新手用户

- 打开应用直接进入项目，不出现账号密码页。
- Leoapi 入口保留在侧栏与顶栏，进入后首张卡就是“本机原配置”，可理解为恢复官方登录状态。
- 接口新增、启用、恢复都有确认提示；密钥只显示脱敏值。

### 熟练与重度用户

- Claude Code、Codex、OpenCode 的真实最短请求均返回成功。
- Leoapi 保留多地址检测、模型列表读取、模型测速、自动选最快地址、Claude 模型映射和备份恢复。
- OpenCode 自定义 DeepSeek 模型通过 CLI 与应用模型目录双重验证。

### 跨设备用户

- 干净设备模拟中 Claude Code、Codex、OpenCode、Cursor、Gemini、Hermes 六个命令全部被识别。
- 当前机器 Cursor 命令已安装，但官方凭据失效；应用正确显示“凭据不可用，请重新登录”，没有误报“未安装”。
- 不复制本机 Agent 凭据到其他设备，每台设备只读取自己的 CLI 与配置。

### 开发与测试

- TypeScript 类型检查通过。
- ESLint 无错误；仓库原有告警仍存在，但本轮未新增阻断错误。
- 自动化：153 项服务端、11 项桌面端、6 项前端测试全部通过。
- 路由测试覆盖原配置快照、OpenCode 应用和恢复；更新器测试覆盖版本重置桥接且不会在 1.1.3 循环更新。

### 发布与运维

- 37 个嵌套 Mach-O/原生模块逐一签名并校验。
- Apple 公证状态 Accepted，Gatekeeper 来源为 Notarized Developer ID。
- 启动后 `127.0.0.1:38473/health` 返回 1.1.3；退出后端口释放。
- 热更新实测使用已安装的 1.36.2，从本地等价更新源下载正式 ZIP，Squirrel 替换后自动重启为 1.1.3。

### 安全、隐私与法务

- 服务仅绑定回环地址；桌面能力 token 每次启动生成，不写入报告。
- Provider 密钥未写入源码、测试、截图或日志，备份和 Provider 存储保持 `0700/0600` 权限。
- GitHub 私有 Release token 使用系统安全存储；子进程环境会清除 GitHub token。
- 品牌和界面已统一为 leocodebox/Leoapi；许可证与 NOTICE 中依法要求保留的第三方归属没有删除。

### 可访问性与视觉

- 模态层有 `role=dialog`、标题、关闭按钮、Esc 关闭和 iframe 标题。
- 浅色、深色、跟随系统三种模式均实测；主界面和 Leoapi 不再出现一明一暗。
- 固定顶栏和可滚动内容区没有文字重叠；启动诊断默认折叠。

## 发现并修复的问题

1. **高危：OpenCode 自定义模型被旧本地缓存覆盖。** 通过配置模型优先级、文件指纹和应用事件硬刷新修复。
2. **高危：热更新下载成功但旧版本无法安装降版本号包。** 初次真实测试在 Squirrel 阶段复现；增加一次性兼容更新版本和 macOS 兼容构建号后，重新签名、公证并完成真实安装闭环。
3. **中危：Leoapi 返回按钮没有动作。** 改为主应用模态层后消除跨 BrowserView 返回依赖。
4. **中危：只能切入自定义接口，不能恢复原登录配置。** 增加不可变原配置快照、恢复路由和原生配置卡。
5. **中危：启动过程暴露后台日志并闪旧启动器。** 改为品牌启动页，诊断折叠，窗口延后显示。
6. **低危：原配置与已选择自定义卡可能同时高亮。** 收敛为单一当前状态。

## 剩余限制

1. 正式包当前仅提供 Apple 芯片版本；Intel Mac 需要单独构建或 universal 包。
2. 仓库和 Release 为私有，应用内检查更新仍需每台设备配置只读 GitHub token；这是 GitHub 私有资产访问限制。
3. 当前机器的 Cursor Agent 官方登录已失效，需要用户执行 `cursor-agent login`；应用本身已正确识别和提示该状态。
